# Проверка розыгрыша Claude · Mastodont AI Hub

Публичный верификатор честности розыгрыша Claude Pro / Claude Code Max на канале [ИИ Мастодонт](https://www.youtube.com/@aiBoomb).

## Как проверить что розыгрыш не подделан

1. **До розыгрыша** организатор публикует в [Telegram-канале](https://t.me/+JULf3S7rEBg2ZTVi) два хэша:
   - `commit_hash` — SHA-256 от секретного seed'а
   - `db_hash` — SHA-256 от списка захэшированных email-адресов участников
2. **После розыгрыша** организатор публикует:
   - `seed` (раскрытый)
   - `proof.json` — полный файл доказательства

3. **Ты** скачиваешь `proof.json`, открываешь [verifier.html](./verifier.html), вставляешь — скрипт прогоняет алгоритм и показывает 3 галочки:
   - ✅ Seed соответствует опубликованному `commit_hash` (организатор не подменил seed)
   - ✅ База участников не менялась (`db_hash` совпадает)
   - ✅ Алгоритм воспроизводит тех же победителей

Если хотя бы одна галочка красная — розыгрыш подделан.

## Участвовал ли я лично?

Во вкладке **"Проверь участвовал ли ты"** вставь свой email. Он хэшируется локально в браузере (никуда не отправляется) и сравнивается с хэшами в базе. Если нашёлся — ты участвовал, видно сколько билетов у тебя было.

## Алгоритм

Детерминированный, воспроизводим в любом языке:

```
1. entries = sort(participants by email.lower() asc)
   где каждый entry = { email_hash: SHA-256(email), weight: 1 + bonus_count }

2. для каждого раунда N (0, 1, 2, ... 5):
     h = SHA-256(seed + ":" + N)
     target = int(h[0:16], 16) mod total_weight
     cumulative = 0
     for entry in entries:
       cumulative += entry.weight
       if target < cumulative:
         winner = entry
         break
     entries.remove(winner)

3. Раунды 0-4 = Claude Pro, раунд 5 = Claude Code Max
```

## Почему это честно

Схема называется **commit-reveal** и используется в криптографии для честной жеребьёвки:

- Организатор **не может** подменить seed после розыгрыша — его хэш опубликован заранее и совпадёт только с исходным seed'ом (найти другой seed с таким же хэшем практически невозможно, SHA-256)
- Организатор **не может** добавить/удалить участника после публикации `db_hash` — любое изменение сломает хэш
- Любой может сам прогнать алгоритм и получить тех же победителей из того же seed'а
- Email'ы не публикуются (только их хэши) — каждый участник узнаёт себя, но чужие email не утекают
